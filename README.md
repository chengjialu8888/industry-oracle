# 🔮 Industry Oracle — Deep Industry Research Accelerator

**Compress a month of industry research into one hour.**

**把一个月的行业研究压缩成一小时。**

---

> Most "industry research" produces 50-page reports that tell you nothing you couldn't find on Google. Industry Oracle does the opposite — it extracts the deep cognition that insiders understand but never say publicly, the kind that usually takes a founder two years and hundreds of customer conversations to develop.
>
> 大多数"行业研究"产出的是 50 页谁都能 Google 到的废话报告。Industry Oracle 反其道而行——它提取的是行业内所有人都懂、但永远不会公开说的底层认知，那种通常需要创始人花两年、聊几百个客户才能真正摸透的东西。

## The Problem / 痛点

You've spent weeks collecting industry reports, competitor decks, founder interviews, and market data. You have 200 browser tabs open. And yet:

你花了几周收集行业报告、竞品资料、创始人访谈、市场数据，浏览器开了 200 个标签页，然而：

- Your analysis sounds like every other consulting deck — correct but useless / 你的分析看起来和所有咨询报告一样——正确但无用
- You can't tell which competitor claims are real vs. capital-fueled narratives / 你分不清哪些竞品的说法是真的，哪些是资本催出来的叙事
- Your assumptions feel solid until an investor asks one question that unravels everything / 你的假设感觉很扎实，直到投资人问了一个问题就全线崩塌
- You don't know what you don't know / 你不知道自己不知道什么

## The Solution / 解法

Industry Oracle is a 4-step structured research process that works like having a world-class research team, five top-tier investors, and a ruthless debate partner — all in one session.

Industry Oracle 是一个四步结构化研究流程，相当于同时拥有一个顶级研究团队、五个一线投资人、和一个毫不留情的辩论对手。

### Step 0 → Material Intake & Gap-Fill / 材料接收与信息补全

Upload whatever you have — PDFs, Feishu docs, web links, pitch decks. The system reads everything, builds an internal knowledge index, identifies critical information gaps, and **proactively searches** to fill them. No more "garbage in, garbage out."

上传你手头所有材料——PDF、飞书文档、网页链接、BP。系统逐一读取，建立内部知识索引，识别关键信息缺口，并**主动搜索补全**。不再"垃圾进，垃圾出"。

### Step 1 → Industry Panoramic Analysis / 行业全景分析

Not a summary. A **paradigm-aware, demand-validated** deep map of the industry:

不是总结。而是一份**范式感知、需求验证**的行业深度认知图谱：

- **Paradigm Assessment**: Is the industry stable, cracking, or mid-migration? This single judgment changes everything downstream. / **范式判断**：行业处于稳定期、裂缝期还是迁移期？这一个判断改变下游所有分析。
- **Competitor Deep Cognition**: Not "Company A is strong in technology" — but what customers know about them but never say out loud. / **竞品深度认知**：不是"A 公司技术很强"——而是客户都知道但不说出口的事。
- **Demand Truth**: Which demand is real (validated by retention + WTP) vs. which is capital-fueled theater? / **需求真相**：哪些需求是真的（有留存率和付费意愿验证），哪些是资本催出的戏？

### Step 2 → Core Assumption Deconstruction / 核心假设拆解

Identifies the **3 assumptions** that, if wrong, would rewrite the entire industry landscape. For each one:

识别出**三个核心假设**——一旦错误，整个行业格局将被改写。对每个假设：

- Supporting evidence from your materials / 来自材料的支持证据
- Specific, observable collapse conditions (not "if things change") / 具体的、可观测的崩塌条件（不是"如果情况变了"这种废话）
- What the post-collapse world looks like / 崩塌后的世界是什么样
- Early signals it's already weakening / 它已经在松动的早期信号

### Step 3 → Investor Pressure Test / 投资人压力测试

Simulates sharp challenges from **5 distinct investor archetypes**:

模拟**五种投资人思维模型**的尖锐质疑：

| Persona | Lens | 视角 |
|---------|------|------|
| **Peter Thiel** | Contrarian monopoly — is this 10× or 10%? | 逆向垄断——这是 10 倍还是 10%？ |
| **Paul Graham / YC** | User obsession — do 100 people love it? | 用户痴迷——有 100 个人真的爱它吗？ |
| **Charlie Munger** | Inversion — how will this die? | 逆向排除——这会怎么死？ |
| **Marc Andreessen** | Tech leverage — why now, not 2 years ago? | 技术杠杆——为什么是现在而不是两年前？ |
| **Industry Veteran** | Operational reality — show me the details | 行业实操——细节拿出来看看 |

### Step 4 → Multi-Round Debate / 多轮辩论

Roles reverse. **You challenge the analysis, and the system helps you forge every argument to its strongest version.** No hand-waving. No "the truth is somewhere in the middle." Every surviving argument earns its place through fire.

角色反转。**你来挑战分析，系统帮你把每个论点打磨到最强版本。** 不允许含糊其辞。不允许"真相在中间"这种废话。每一个幸存的论点都是从火里烧出来的。

## Quick Start / 快速开始

### Installation / 安装

1. Clone this repository / 克隆此仓库：
   ```bash
   git clone https://github.com/chengjialu8888/industry-oracle.git
   ```

2. In your Mira skill directory, add the `SKILL.md` file and the `references/` folder / 在你的 Mira 技能目录中，添加 `SKILL.md` 文件和 `references/` 文件夹

3. Start a conversation with any of these triggers / 用以下任一触发词开始对话：
   - "帮我做一个行业深度研究"
   - "Analyze this industry for me"
   - "做个竞品深度分析"
   - "Help me prepare for investor Q&A"
   - "拆解一下这个赛道的核心假设"

### Usage / 使用方式

```
You: 帮我深度分析 [行业/赛道]，这是我收集的材料 [上传文件]

Oracle: [Step 0] 材料接收完毕，发现以下信息缺口...已搜索补全...
        → 确认后进入 Step 1？

You: 确认，开始吧

Oracle: [Step 1] 生成飞书文档 → 行业全景分析
        → 2-3 句核心发现 + 等待你的反馈

...（逐步推进，每步暂停确认）
```

## Repository Structure / 仓库结构

```
industry-oracle/
├── README.md                          ← You are here / 你在这里
├── SKILL.md                           ← Main skill file (always loaded) / 主技能文件（始终加载）
└── references/                        ← On-demand reference files / 按需加载的参考文件
    ├── frameworks.md                  ← 7 analysis frameworks / 7 种分析框架
    ├── investor-personas.md           ← 5 investor thinking models / 5 种投资人思维模型
    └── examples.md                    ← Good vs Bad output calibration / 输出质量校准示例
```

**Progressive Disclosure Architecture**: `SKILL.md` is always in context. Reference files are loaded only when needed (e.g., `frameworks.md` loads during Step 2, `investor-personas.md` loads during Step 3), keeping the context window efficient.

**渐进式加载架构**：`SKILL.md` 始终在上下文中。参考文件仅在需要时加载（如 `frameworks.md` 在 Step 2 加载，`investor-personas.md` 在 Step 3 加载），保持上下文窗口高效。

## The Bar We Hold / 我们坚持的标准

Every insight must pass this test:

每一个洞察都必须通过这个测试：

> **"Would a 3-year industry veteran say: 'Yes — but I've never seen anyone articulate it this clearly'?"**
>
> **"一个在这个行业干了三年的人会说：'对，但我从来没见人这么清楚地说出来过'吗？"**

If the answer is no, it gets cut. Correct-but-obvious is the enemy.

如果答案是否，就砍掉。正确但显而易见的东西是最大的敌人。
